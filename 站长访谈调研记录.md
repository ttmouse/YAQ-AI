# 一起安 · 站长安全监管工作台 用户调研记录

> **调研对象**：杨站长（街道/镇级安全生产分管领导）  
> **调研日期**：2025年  
> **调研目的**：理解站长每天查看安全监管工作执行情况的真实管理判断逻辑，为"一起安"智能体能力设计提供依据  
> **核心场景**：站长/分管领导每天查看站内安全监管工作执行情况

---

## 一、每天最想先看到哪些工作情况

杨站长最需要优先掌握的是 **整体安全态势、关键风险闭环、核心任务进展** 三大方面：

### 1.1 整体安全态势

- 风险底数、风险区域是否变化
- 新增了哪些风险隐患



### 1.2 关键风险闭环

主要是重大隐患的闭环处置情况，重点关注：

- 人员密集场所消防通道堵塞
- 住宅小区自动化消防设施失效
- 电动自行车违规停放

要求：未闭环的需要能看到列表，或点链接进去查看详情。

### 1.3 核心任务进展

- **政府/村社侧任务完成情况**：日常任务、专项任务是否"时间过半、任务过半"
- **企业主体责任落实情况**：隐患自查自纠、员工培训等
- **企业自查 vs 政府检查对比**：将政府对企业检查发现的隐患与企业自查发现的隐患进行对比，判断企业是认真在做还是敷衍了事

### 1.4 远期期望：风险预测模块

希望后续开发风险预测模块或模型，当前历史数据量不足。待数据积累后，对历史数据（火灾情况、隐患整改记录、企业类型等多维度）进行分析，预测高风险区域、高发隐患类型及潜在风险点，**变被动处置为主动预防**。

## 二、如何判断"今天有没有问题"

杨站长总结为 **四个步骤**：

### 步骤一：整体判断

通过面上指标判断是否存在异常：

| 指标维度 | 说明 |
|---------|------|
| 隐患发现率和闭环率 | 判断巡查和整改的总体效率 |
| 重大隐患处理进度 | 关注未闭环的重大隐患 |
| 应消站及各村社履职率 | 日常工作、专项工作等任务的完成进展，是否时间过半、任务过半 |
| 企业主体责任落实情况 | 隐患自查上报率、员工培训完成率、应急逃生演练完成情况等 |

通过这些指标判断昨天或前一段时间是否有问题，有问题则进入深入排查。

### 步骤二：定位问题

**当隐患闭环率较低时：**

- **看类型**：分析是哪一类隐患整改困难最多？是客观条件限制（如企业不配合）还是主观不作为？
- **看区域**：未闭环的隐患集中在哪些区域（如哪条街的沿街店铺或哪个园区的企业）？是否存在区域性的监管真空？
- **看责任**：具体是哪个片区（绕城内/绕城外的哪个组长、组员负责的，或哪个区域站负责的区域）？是否存在履职不到位的情况？

**当专项行动进度落后时：**

- 查看是哪个环节拖了后腿？是检查任务分配不均，还是隐患整改反馈延迟？或是哪个专家水平不行？

**当企业风险上升时（如系统研判出"企业风险上升"变红）：**

- 调取风险上升企业的详细信息，查看其近期的自查记录、隐患数量变化、人员变动等，判断是偶发事件还是系统性风险增加。

### 步骤三：盯牢高风险点（或"面"）

重点关注对公共安全构成直接威胁的高风险点：

1. **重大隐患回头看**：确认是否已采取临时管控措施，是否制定了详细的整改方案和时间表，责任人是谁
2. **重点单位和重点企业**（高层住宅、商业综合体、危化品企业）：特别标注企业单位落实主体责任的相关数据，根据数据判断责任落实情况
3. **异常数据调查**：对于某个区域或某种类型隐患突然激增的情况，深入调查背后原因，判断是真实风险爆发还是数据统计误差

### 步骤四：解决问题

**判断为"没问题"** → 风险可控，加强日常巡查、宣传，做"防"的工作即可。

**判断为"有问题"**，分两类：

| 严重程度 | 表现 | 处置方式 |
|---------|------|---------|
| 较轻 | 个别指标轻微下滑、少量隐患未闭环 | 督促相关责任人落实工作；希望通过平台留言方式提醒相关责任人加快进度 |
| 较严重 | 任务完成情况严重落后、多个重大隐患待处理 | 组织开会讨论研究整改方案，或到现场查看，再明确整改时限和责任人 |

## 三、异常排查顺序

杨站长梳理了 **六步排查流程**，并强调可根据问题紧急性灵活调整：

### 第1步：确认异常

- 通过异常的具体表现（进度滞后、隐患未闭环等），判断影响范围、紧急程度
- 进行问题初步定级
- 涉及重大隐患或核心任务进度受阻 → **视为紧急事件**

### 第2步：按等级筛选问题

- 优先处理高风险问题
- 红色预警的重大隐患和对整体目标有重大影响的任务优先处理
- **示例**：春节期间外省连续发生涉烟花爆竹的重大生产事故 → 将排查涉烟花爆竹的问题放在重要的专项任务中
- 原则：类似质量管理和风险管理中**优先处理高影响、高紧急度问题**的策略

### 第3步：定位具体条线和专项任务

- 定位到具体条线（如企业安全组或场所安全组等）
- 定位到某个专项任务（如烟花爆竹排查、高层小区排查等）
- 通过梳理判断是**系统性偏差**还是**个别问题**

### 第4步：定位区域

- 将问题定位到具体片区（绕城内外、物流等）、村社网格、工业园区或哪条街哪个综合体
- 有助于识别是否存在区域性的管理薄弱环节或外部环境因素影响

### 第5步：核查责任主体与人员履职

- 核查相关责任主体（如企业自查情况）和具体责任人（如组员、专家、企业安管员等）的履职记录
- 确认是否存在人员失职、宣传不足或企业主体责任不落实的情况
- 分析操作层面和人员因素

### 第6步：定位具体企业场所与隐患

- 查看具体的企业名单和隐患详情
- 分析典型个案，查找根本原因（企业自身管理缺陷、设备老化或流程漏洞）
- 为后续制定精准的整改措施提供依据

### 补充原则

- 以上步骤可根据问题的紧急性进行灵活调整，设置太死可能会影响工作
- 无论问题大小（一般隐患、重大隐患），都应**跟踪至整改完成**，形成完整闭环
- 对于反复出现或系统性的异常，应深入分析，考虑优化流程、加强培训或调整平台功能，以实现**持续改进**

## 四、问题定位的颗粒度要求

杨站长认为颗粒度需要根据问题性质、严重程度和所处阶段灵活决定，分三个层次：

### 层次一：宏观 — 整体风险评估

- 先将整个街道视为一个整体进行风险评估
- 初步判断问题可能集中在哪些"线"上（如企业安全/消防安全/森林防火等）
- 快速确定主攻方向

### 层次二：中观 — 定位到具体负责人

- 明确问题所属业务线后，将责任落实到具体个人或团队
- **涉及的角色**：片区负责人（一般是组长，如沈泽东、孙中振等）、专家、区域站负责人、行业监管科室、村社
- 目的：压实责任、明确问题处置与责任划分

### 层次三：微观 — 聚焦到经营主体或具体隐患

- 对于反复出现或较为严重的隐患，深入到具体的企业、场所（如出租房、商场等）
- 查清隐患的具体情况、涉及的具体人员
- 进行精准、彻底的整治

### 核心原则

> 安全生产、消防安全、防灾减灾是压实"责任"的工作，任何问题都必须明确至少一名责任人，这是确保执行力的基础。对于重大或复杂的问题，要将责任和措施分解到最具体的单元——无论是某个具体片区、企业，还是某位具体人员，直至问题得到根本解决。

## 五、定位问题后的行动措施

杨站长将行动分为 **内部管理（街道/村社人员）** 和 **外部管理（经营主体）** 两条线，目标是"闭环管理"、"压实责任"。

### 5.1 内部管理：针对街道/村社层面人员

按力度递进的处置路径：

| 级别 | 行动 |
|------|------|
| 轻微 | 通过微信（或浙政钉）、电话了解情况，督促进一步落实 |
| 中等 | 将张义、国生、组长、村社相关人员等叫到办公室谈话 |
| 较重 | 一键提醒应消站、区域站、第三方专家、村社的人员履职 |
| 严重 | 对上述人员进行约谈，甚至对第三方服务单位进行考核（扣钱） |

### 5.2 外部管理：针对经营主体

按力度递进的处置路径：

| 级别 | 行动 |
|------|------|
| 第1级 | 安排组长带队（带上专家；低风险单位则叫上村社和区域站）深入企业、场所进行现场督促指导，了解具体原因，帮助解决实际问题，落实隐患整改 |
| 第2级 | 杨站长及张义、国生出面对企业、场所进行约谈，施加压力、强调责任 |
| 第3级 | 要求企业存在隐患的区域停业整顿 |
| 第4级 | 应消站联合综合执法、市场监管等部门开展联合检查，进一步施压 |
| 第5级 | 上报应急局或消防大队建议立案处罚，处罚情况通报给企业、场所 |

### 5.3 系统性改进

- **内部管理方面**：如果发现此类问题并非个别现象而是普遍存在，必须进行复盘分析，检查工作流程、责任分工等是否存在漏洞，针对性优化工作标准，加强过程管控
- **经营主体方面**：以此为契机，在全街道范围内开展一次专项整治行动，对同类场所、同一类型的隐患进行专项排查，防患于未然

> **总结**：既要解决当前的具体问题，也要通过处理这些问题，倒逼人员提升工作能力、压实责任、完善制度，最终实现整体工作水平的提升。

## 六、智能体每天应主动告知的内容

杨站长认为智能体的作用主要是**提升决策效率和风险防控能力**，最希望智能体主动告知以下内容（按优先级排列）：

### 优先级 1：重大风险整改闭环情况

**形式**：每日《重大隐患整改日报表》

**内容**：
- 当天新发现的所有重大隐患，包括具体位置、涉及单位、隐患描述
- 昨日及历史遗留重大隐患的整改进展、当前状态（已整改/部分整改/未整改）及具体负责人
- 对逾期未完成整改的事项进行高亮标注，支持一键发起督办提醒

**目的**：确保第一时间掌握最关键的风险点，并监督整改措施的落实，避免小问题演变成大事故。

### 优先级 2：重点监管企业/场所的异常预警

**形式**：实时数据监测 + 主动预警

**内容**：
- 对纳入重点监管名录的企业、场所进行实时数据监测
- 主动预警异常情况

**目的**：对高风险单位实现精准、动态的监控。

### 优先级 3：一线人员工作效能分析报告

**形式**：每日《工作效能简报》

**内容**：
- 应消站检查人员（包括专家）当日的任务完成率、隐患发现数（特别是重大隐患）
- 区域站、村社的数据到"站"和"村社"级别即可，不需要精确到每一位人员
- 近期整改通知书的下发与复查闭环率对比
- 对工作效率异常低下或隐患发现率明显偏低的人员进行提示

**目的**：通过数据化方式客观评估团队工作状态，及时发现问题，有针对性地进行指导、培训或约谈，提升整个队伍的战斗力和责任心。

### 通用要求

- 报表、简报无固定格式，要求**简单、易懂、突出重点**
- 报表需有**前后对比分析**（如上周一报表中的重大隐患是 3 个，一周以后还是这 3 个，可能存在人员履职不到位的问题）

## 附录：关键管理理念提炼

| 理念 | 说明 |
|------|------|
| 闭环管理 | 任何问题都必须跟踪至整改完成，形成完整闭环 |
| 压实责任 | 任何问题都必须明确至少一名责任人 |
| 颗粒度分层 | 宏观（整街道）→ 中观（负责人/条线）→ 微观（企业/隐患） |
| 优先级排序 | 高风险、高紧急度问题优先处理 |
| 数据对比 | 企业自查 vs 政府检查、前后时间对比、区域横向对比 |
| 系统性改进 | 从个案中提炼共性问题，优化流程和制度 |
| 灵活应变 | 排查步骤可根据紧急性灵活调整，不设太死 |
